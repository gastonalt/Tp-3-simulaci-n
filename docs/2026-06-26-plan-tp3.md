- Un sistema de colas `M/M/1`, con analisis teorico, simulacion en Python e implementacion en AnyLogic.

Para ambos modelos se deben presentar medidas finales, evolucion temporal, graficos, marco teorico, formulas y comparacion entre tres fuentes:

## Fase 2 - Marco teorico del modelo `M/M/1`

1. Documentar supuestos del modelo:
   - Arribos Poisson.
   - Tiempos entre arribos exponenciales.
   - Tiempos de servicio exponenciales.
   - Un servidor.
   - Disciplina FIFO.
2. Definir variables:
   - `lambda`: tasa de arribo.
   - `mu`: tasa de servicio.
   - `rho = lambda / mu`: utilizacion.
3. Para cola infinita y regimen estable, documentar:
   - Condicion de estabilidad: `rho < 1`.
   - Promedio de clientes en sistema: `L = rho / (1 - rho)`.
   - Promedio de clientes en cola: `Lq = rho^2 / (1 - rho)`.
   - Tiempo promedio en sistema: `W = 1 / (mu - lambda)`.
   - Tiempo promedio en cola: `Wq = lambda / (mu * (mu - lambda))`.
   - Utilizacion del servidor: `rho`.
4. Aclarar que para `lambda >= mu` el modelo `M/M/1` con cola infinita no tiene estado estacionario.
5. Para cola finita, definir:
   - Tamanio maximo de cola `B`.
   - Capacidad total del sistema `K = B + 1`.
   - Probabilidad de estado `P_n`.
   - Probabilidad de denegacion de servicio `P_K`.
   - Tasa efectiva `lambda_eff = lambda * (1 - P_K)`.
6. Documentar como se calculara la probabilidad de `n` clientes en cola.

## Riesgos y cuidados importantes

- Para `M/M/1` con cola infinita, los casos `lambda >= mu` no tienen regimen estable. Deben analizarse con cuidado y explicarse explicitamente.
- En cola finita, incluso con `lambda >= mu`, el sistema puede analizarse porque el exceso de demanda se transforma en rechazos.
- La capacidad de cola pedida no es igual a la capacidad total del sistema. Si la cola tiene tamanio `B`, entonces el sistema completo tiene capacidad `K = B + 1`.
- Las comparaciones deben usar las mismas unidades en teoria, Python y AnyLogic.
- Las corridas deben usar semillas registradas para poder reproducir resultados.
- Los graficos temporales deben indicar claramente si muestran una corrida individual o un promedio entre corridas.
- Si se usa periodo de calentamiento, debe documentarse y aplicarse de forma consistente.
